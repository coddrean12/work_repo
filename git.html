Introduction:

Git-hub is an online server space that allows for collaboration and allows you to keep track of your changes to codes.

Git-hub allows people to collaborate on a single project. Whenever you make changes to codes, it will keep it updated.

You can do trial and error with your codes and still go back to your previous codes.

Git commands:

- Git clone: Clones a remote repository to a local repository. Brings what is on git-hub to your system.

- Git add: Adding/including all folders and files under something so that you can push everything together.

- Git commit: Saves the files so that when you want to push, the push will know what to push.

- Git push: Sends the codes to the server.

- Git status: To know if your folder is a git repository

- Git remote add origin: Specifies what should be pushed to your server.

- Git remote -v: Shows you where you are fetching or pushing from.

- Git pull: Makes sure your remote repository is in sync with your local repository.

- Git log: Helps you to see your commit history. Shows you interactions that happen on your git hub account.

How to open a folder in git bash:

Do cd ../.. to come out of a previous folder. Do ls to check what is inside your c drive, then do cd and the name of the folder you want to open.

Note:

If git-hub asks for who you are, the authentication to tell it are; your username, your password, your email or your access token.
